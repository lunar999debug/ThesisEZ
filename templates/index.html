ThesisEZ

一键把"裸奔"Word 草稿变成符合学校规范的毕设文档

AI 解析结构 + Pandoc 渲染引擎 · 不用学 Markdown · 不用拆文件

📦 源代码 GitHub

它解决什么问题

学位论文的"内容"和"格式"是两件事。学校发一份《格式要求.pdf》和一份 Word 模板让你自己对齐字号、行距、目录域、题注编号——多数同学就是在最后一周对着这些反复磨。

ThesisEZ 让你写只管内容的草稿（字号字体随便），AI 解析结构，渲染引擎用学校 Word 模板套出符合规范的最终 docx。

默认带 SJTU 中文模板（因为我自己是 SJTU），但模板可以替换成任意学校的 .docx。

⚙️ 怎么实现的

  • AI 解析层（aichuli.py）：DeepSeek + 硬正则兜底，识别每段是 h1/h2/h3/正文/题注/参考文献
  • 渲染引擎层（render.py）：Pandoc + python-docx，套学校模板做后处理（三线表、题注、SEQ 域、分节符、目录）
  • 多重防御：AI 错了 5% 也不能让用户拿到看似正确实际错的输出 → 严格 Prompt + 硬正则兜底 + 状态机限定生效区域

🔒 隐私说明

  • 本 Demo 只在内存里处理你的文件，渲染完立即销毁，不存任何副本
  • 但本 Demo 仅供评审/试用，请不要上传你真正未发表的毕设草稿到任何陌生服务器（包括本站）
  • 真正生产用法：clone GitHub 仓库，本地跑
